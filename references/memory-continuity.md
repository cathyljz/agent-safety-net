# Memory continuity

## What problem this solves
Without continuity, users must restate:
- personal preferences
- prior decisions
- unfinished tasks
- project context

That makes agents feel stateless and unreliable.

## Practical fix
Separate memory into two layers:
- long-term memory for durable preferences and decisions
- active-task state for the current job only

Keep both small and curated.

## Good safeguard
Only store what helps future execution:
- user preferences
- recurring workflows
- stable constraints
- key decisions
- open blockers

Do not dump entire conversations into memory.

## Why beginners care
People want the feeling of “you remember how I work”, not “please paste everything again”.
