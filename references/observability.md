# Observability

## What problem this solves
Users lose trust when they cannot answer three simple questions:
- what is the agent doing now?
- why did it stop?
- what should happen next?

## Practical fix
Expose short, useful status:
- stage
- latest success
- current blocker
- next action
- risk level if relevant

## Good safeguard
Prefer concise human-readable updates over dense technical traces.
Technical detail should be available when needed, but not be the default.

## Why beginners care
Good observability makes the agent feel dependable instead of mysterious.
